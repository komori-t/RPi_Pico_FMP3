１．LPC55S69-EVKターゲット依存部の概要

LPC55S69-EVKターゲット依存部（GNU開発環境向け）は，TOPPERS/ASP3カーネル
を，NXP LPC55S69を搭載したLPC55S69-EVKボード上で動作させる環境を構築する
ためのものである．なお，LPC55S28-EVKにおいても動作を確認している．

1.1 対応するターゲットシステムとターゲット略称

LPC55S69-EVKターゲット依存部（GNU開発環境向け）の動作確認は，実機を用いて
行っている．LPC55S69-EVKに関する情報は，以下のウェブサイトにある．

	https://www.nxp.com/products/processors-and-microcontrollers/
	arm-microcontrollers/general-purpose-mcus/
	lpc5500-cortex-m33/lpcxpresso55s69-development-board:LPC55S69-EVK

プログラムをフラッシュメモリに書き込んで実行する方法（ROM実行）のみを
サポートしている．

ターゲット略称等は次の通り．

	ターゲット略称：lpc55s69evk_gcc
	システム略称：lpc55s69evk
	開発環境略称：gcc

1.2 ターゲット依存部の構成

LPC55S69-EVKターゲット依存部（GNU開発環境向け）は，チップ依存部として
LPC5500チップ依存部（GNU開発環境向け）を，コア依存部としてARM_Mコア
依存部（GNU開発環境向け）を使用している．

ターゲット依存部（targetディレクトリ）およびターゲット依存部の共通部
（archディレクトリ）のディレクトリは次の通り．

	target/
		lpc55s69evk_gcc/	LPC55S69-EVKターゲット依存部

	arch/
		arm_m_gcc/common/	ARM_Mコア依存部
		arm_m_gcc/lpc5500/	LPC5500チップ依存部
		arm_m_gcc/doc/		ARM_M依存部に関するドキュメント
		gcc/				GCC開発環境依存部

1.3 依存している個別パッケージのバージョン番号

LPC55S69-EVKターゲット依存部（バージョン 3.5.0）の個別パッケージが
依存している個別パッケージと，動作確認を行ったバージョンは次の通り．

	個別パッケージの名称	バージョン	個別パッケージファイル名
	------------------------------------------------------------------
	ターゲット非依存部		3.5.0		asp3-3.5.0.tar.gz

1.4 開発環境／デバッグ環境と動作確認条件

開発環境として，以下のURLからプリビルド版をダウンロードすることができ
るGNU ARM Embedded Toolchainを用いている．

	https://developer.arm.com/open-source/gnu-toolchain/gnu-rm

動作確認を行ったバージョンは次の通り．

	gcc: 8.3.1 20190703
	binutils（objcopy，objdump）：2.32.0.20190703

デバッグ環境として，オンボードのCMSIS-DAPから，pyOCDを経由して，GDB
を用いる環境で動作確認を行っている．動作確認を行ったデバッグ環境のバー
ジョンは次の通り．ただし，pyOCDによるFlashへのプログラムイメージの書
き込みはRevision 0Aのチップのみで動作を確認している．

	gdb: 9.0.50.20191101-git
	pyocd: 0.23.0

GUIベースの開発環境／デバッグ環境として，NXP MCUXpressoを用いるこ
とができる．この場合，LPC55S69 SDKがMCUXpressoにインストールされ
ている必要がある．動作を確認したSDKのバージョンは以下の通り．

	LPC55S69 SDK: 2.7.1

MCUXpressoの利用方法

MCUXpressoディレクトリをワークスペースとして起動する．

File -> Import -> General -> Existing Projects into Workspace
を選択する．

Select root directory にMCUXpressoディレクトリを指定し，sample
にチェックを入れてFinishをクリックする．

プロジェクトツリー内のconfigure.launchを右クリックし，Run As -> configure
を選択することでコンフィギュレーションが行われる．

Project -> Build Project を実行することでビルドが行われる．

プロジェクトツリー内のsample.launchを右クリックし，Debug As -> sample
を選択することでデバッグが行える．

1.5 メモリマップ

以下のメモリマップを想定している．

	0x00000000 - 0x00098000：内蔵フラッシュメモリ（608KB）
	0x20000000 - 0x20044000：内蔵SRAM（272KB）
	0x04000000 - 0x04008000：内蔵実行用SRAM（32KB）
	0x40000000 - 0x40110000：I/O領域（1.088MB），予約領域を含む

メモリマップを変更する場合には，lpc55s69.ldを修正する必要がある．


２．ターゲット定義事項の規定

LPC55S69-EVKターゲット依存部は，ARM_Mコア依存部とLPC5500チップ
依存部を用いて実装されている．ARM_Mコア依存部およびLPC5500チップ
依存部におけるターゲット定義事項の規定については，「ARM_M依存部 
ユーザーズマニュアル」を参照すること．


３．ドライバ関連の情報

3.1 タイマドライバ

高分解能タイマは，LPC55S69が持つ4チャンネルのStandard Timer/Counter
の内CTIMER0を用いて実現している．

3.2 シリアルインタフェースドライバ

シリアルインタフェースドライバは，LPC55S69が持つ8ポートのUSARTインタ
フェースをサポートしている．

各ポートは，以下の通りに設定している．

	ボーレート：115200bps
	データ：8ビット
	パリティ：なし
	ストップビット：1ビット


3.3 システムログの低レベル出力

システムログの低レベル出力は，シリアルインタフェースドライバのポート0と
同じチャネル（すなわち，USARTのチャネル0）に対して，ポーリングにより文字
を出力する方法で実現している．

用いるポートを変更する場合には，target.cdlを修正する必要がある．


４．システム構築手順と実行手順

LPC55S69用のASP3カーネルおよびサンプルプログラムを構築する手順は，
「TOPPERS/ASP3カーネル ユーザーズマニュアル」の「３．クイックス
タートガイド」の章に記述されている通りである．

4.1 ドライバのインストール

LPC55S69-EVKのUSBポートとPCを接続しても，LPC55S69-EVKがシリアル
ポートとして認識されない場合には，USBシリアルドライバをインストール
する必要がある．

Windows向けのドライバは，以下のURLにある．

	https://developer.mbed.org/handbook/Windows-serial-configuration

MacOS X用のUSBシリアルドライバは，例えば，以下のURLにあるものを使用する
ことができる．

	https://www.ftdichip.com/Drivers/VCP.htm
